ia-bot-ahmed Windows'ta dogrudan calisabilir, ancak en iyi deneyim icin [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install) kullanmanizi oneriyoruz. WSL, ia-bot-ahmed ozellikleriyle sorunsuz calisan bir Linux ortami saglar.

:::tip[Neden WSL?]
WSL, daha iyi dosya sistemi performansi, tam terminal destegi ve ia-bot-ahmed'un dayandigi gelistirme araclariyla uyumluluk sunar.
:::

---

## Kurulum

<Steps>

1. **WSL'i yukleyin**

   Henuz yuklemediyseniz resmi Microsoft rehberini kullanarak [WSL kurulumunu](https://learn.microsoft.com/en-us/windows/wsl/install) yapin.

2. **ia-bot-ahmed'u WSL icine yukleyin**

   WSL hazir olduktan sonra WSL terminalini acin ve ia-bot-ahmed'u [kurulum yontemlerinden](/docs/) biriyle yukleyin.

   ```bash
   curl -fsSL https://ia-bot-ahmed.app/install | bash
   ```

3. **ia-bot-ahmed'u WSL icinden kullanin**

   Proje dizininize gidin (`/mnt/c/`, `/mnt/d/` vb. ile Windows dosyalarina erisebilirsiniz) ve ia-bot-ahmed'u calistirin.

   ```bash
   cd /mnt/c/Users/YourName/project
   ia-bot-ahmed
   ```

</Steps>

## Masaüstü uygulaması + WSL sunucusu

ia-bot-ahmed Desktop kullanmak isteyip sunucuyu WSL'de calistirmak istiyorsaniz:

1. **Sunucuyu WSL'de baslatin** ve dis baglantilara izin vermek icin `--hostname 0.0.0.0` kullanin:

   ```bash
   ia-bot-ahmed serve --hostname 0.0.0.0 --port 4096
   ```

2. **Desktop uygulamasini** `http://localhost:4096` adresine baglayin

:::note
Kurulumunuzda `localhost` calismiyorsa bunun yerine WSL IP adresini kullanin (WSL icinde: `hostname -I`) ve `http://<wsl-ip>:4096` adresine baglanin.
:::

:::caution
`--hostname 0.0.0.0` kullaniyorsaniz sunucuyu guvene almak icin `IA_BOT_AHMED_SERVER_PASSWORD` ayarlayin.

```bash
IA_BOT_AHMED_SERVER_PASSWORD=your-password ia-bot-ahmed serve --hostname 0.0.0.0
```

:::

## Web istemcisi + WSL

Windows'ta en iyi web deneyimi icin:

1. **`ia-bot-ahmed web` komutunu PowerShell yerine WSL terminalinde calistirin**:

   ```bash
   ia-bot-ahmed web --hostname 0.0.0.0
   ```

2. **Windows tarayicinizdan** `http://localhost:<port>` adresini acin (ia-bot-ahmed URL'yi yazdirir)

`ia-bot-ahmed web` komutunu WSL'den calistirmak, dosya sistemi erisimi ve terminal entegrasyonunu daha tutarli hale getirirken tarayicidan erisimi korur.

---

## Windows dosyalarına erişin

WSL, tum Windows dosyalariniza `/mnt/` dizini uzerinden erisebilir:

- `C:` surucusu -> `/mnt/c/`
- `D:` surucusu -> `/mnt/d/`
- Ve digerleri...

Ornek:

```bash
cd /mnt/c/Users/YourName/Documents/project
ia-bot-ahmed
```

:::tip
En sorunsuz deneyim icin deponuzu WSL dosya sistemine (ornegin `~/code/` altina) klonlayip ia-bot-ahmed'u orada calistirmayi dusunun.
:::

---

## İpuçları

- Windows suruculerinde duran projelerde ia-bot-ahmed'u WSL icinde calistirin - dosya erisimi daha sorunsuz olur
- Butunlesik bir gelistirme akisi icin ia-bot-ahmed ile birlikte VS Code [WSL eklentisini](https://code.visualstudio.com/docs/remote/wsl) kullanin
- ia-bot-ahmed konfigurasyonunuz ve oturumlariniz WSL ortaminda `~/.local/share/ia-bot-ahmed/` konumunda saklanir
